Gerando buil de produção

Build de DEV

O build será gerado dentro de um diretório chamado dist com 3 arquvos:

  • Index.html - Ele adiciona o import inline do do main.bundle.js e do polyfills.bundle.js
  • Main.bundle.js - Contem todo código fonte da aplicação, incluindo os template html e css
  • Polyfills.bundle.js - Contem o código auxiliar que ajudará o browser a renderizar o código

Para gerar o build de dev temos os seguintes comando:

  1. ng build --target=development --environment=dev
  2. ng build --dev --e=dev
  3. ng build --dev
  4. ng build

Build de produção (build final)

Usado para gerar o build final compactado e em um único arquivo, gera os mesmos arquivos que o build de dev só que com o arquivo main e polyfills minificados , as variaveis são renomeadas em uma única letra para poder ficar mais leve o software e diminuir o tamanho do arquivo, os arquivos css também serão minificados no bundle.

Para gerar o build de produção:

  1. ng build --target=production --environment=prod
  2. ng build --prod --env=prod
  3. ng build --prod
